git 工作区域与文件状态

git的工作区域分为：

文件存在以下几种状态：

  • 未跟踪(Untrack)
  • 未修改(Unmodified)
  • 已修改(Modified)
  • 已暂存(Staged)
  • 已提交

提交时用-m指定提交信息：

提交到暂存区时使用*.txt会把txt结尾的文件提交到暂存区中；使用git add .将当前文件夹下的所有文件都添加到暂存区里边。

使用git log命令查看提交记录。

git reset 回退版本

三种模式：

  1. git reset --soft
  2. git reset --hard
  3. git reset --mixed

三种模式后面加上要回退的版本ID即可。eq.191d9e3 (HEAD -> master) first commit中的191d9e3

使用soft参数回退时，工作区与暂存区的内容不会被清空；使用hard参数，工作区与暂存区都会清空版本后的文件；mixed会清空暂存区的，但不清空工作区的。

回退到上一个版本可以使用HEAD^。

.gitignore

  • 系统或者软件自动生成的文件；
  • 编译产生的中间文件和结果文件；
  • 运行时生成的临时文件；
  • 敏感信息文件。

例如echo access.log > .gitignore忽略access.log这一日志文件。

该文件的匹配规则时从上到下匹配的，空行与以#开头的行会被git忽略掉；

使用标准的Blob模式匹配：*表示任意个字符，？表示单个字符，[]表示匹配列表里的单个字符；**表示任意中间目录；!表示取反
